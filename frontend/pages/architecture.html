System design

End-to-end architecture for local research inference

This project now reflects your actual deployment stack: a browser frontend, Flask inference API, and a fine-tuned XLM-RoBERTa token classification model stored in the backend folder.

Frontend interface

Multi-page website built with HTML, CSS, and JavaScript. Includes login state, dashboard analytics, documentation, contact form, and an analysis workspace.

Flask API layer

Loads the tokenizer and model from `backend`, performs inference, exposes `/health` and `/analyze`, and returns word-level entities aligned with your notebook logic.

XLM-R inference core

`XLMRobertaForTokenClassification` predicts BIO labels for code-mixed Roman Urdu + English text, then the backend merges them into clean entity spans.

Inference pipeline
1

Client request

The analyzer page submits raw text through `frontend/js/api.js` to `POST /analyze`.

2

Colab-aligned preprocessing

Flask normalizes the text, lowercases it, splits by whitespace, and tokenizes with `is_split_into_words=True` to match your notebook prediction style.

3

First-subtoken prediction

Only the first subtoken prediction per word is kept, just like your Colab `predict_sentence()` implementation.

4

Frontend presentation

The UI highlights entities, renders token scores, and stores the result in session-aware local history.

Persistence design

Ready for auth and database history

The current frontend already has the user flows needed for a production-like system. Today, those flows are stored in browser local storage. Later, you can connect them to real backend endpoints and a database.

  • Login session
    A reusable login modal stores user identity and role across pages.
  • Result history
    Each analysis is saved per active session, then surfaced inside the analyzer sidebar and dashboard page.
  • Database migration path
    Replace local storage with backend routes for register/login/history without redesigning the frontend.

Suggested future backend extensions:

POST /auth/login, POST /auth/register, GET /history, POST /history
