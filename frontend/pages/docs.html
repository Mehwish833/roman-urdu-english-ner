CogniTag project guide

CogniTag is a browser-based research interface for testing a local code-mixed NER model. It combines a Flask backend with a static frontend that runs well in VS Code using a simple local server.

Run the project

Daily startup steps

Terminal 1 (backend)

cd backend

.\run_backend.bat

Terminal 2 (frontend)

cd frontend

python -m http.server 8000

Open browser

http://127.0.0.1:8000/

API

Current backend endpoints

  • GET /health
    Checks whether Flask is up and the model can be loaded.
  • POST /analyze
    Accepts `{ "text": "..." }` and returns processed text, entity spans, token-level predictions, and model metadata.
Frontend

What is functional right now

  • Session login UI
    Users can save a name, email, and role into browser storage and carry that identity across pages.
  • Result history
    Every analysis is stored locally and appears in the analyzer sidebar and dashboard page.
  • Result export
    Analyzed outputs can be exported to JSON for reporting or debugging.
Next phase

How to connect login and history to a cloud database later

The site is already structured for that upgrade. Replace the browser-only storage calls with backend API calls and persist them in global cloud databases like SQLite, PostgreSQL, or MongoDB.
